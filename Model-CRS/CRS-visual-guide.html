Guide

Step-by-step rules for building a Canonical Realm String from an earth system model's embedding and coupling graph.

0
Realm order — always this sequence

Every sort, every tiebreak, uses this fixed order.

Fixed realm ordering
A atmosphere
Ac atmos-chem
Ae aerosol
L land
Li land-ice
O ocean
Ob ocean-biogeo
Si sea-ice
Index: A=0, Ac=1, Ae=2, L=3, Li=4, O=5, Ob=6, Si=7. Lower index = earlier.
1
Build the embedding string

Do this first for every superior host. The result is used in all later steps.

Rule — write node + [children] recursively, realm order
GivenAe embedded in Ac,
Ac embedded in A
A[Ac[Ae]]
GivenOb and Si
both in O
O[Ob][Si]

Identify the coupling graph topology

Apply the first rule that matches. Use embedding strings from Step 1 wherever a node appears.

Rule A
Single superior host — no couplings
GraphOne node, nothing to couple with

Rule B
Full clique — 4 or more hosts, every pair coupled → *{ }
Only for 4+ nodes. A 3-node clique is just a triangle — use Rule C.

Quick reference
Notation symbols
Symbol Meaning Example
[ ] embedded child A[Ac] — Ac inside A
{ } cycle (endpoints coupled) {ALO} — triangle
*{ } full clique, 4+ nodes *{ALLiO} — K4
( ) on cycle chord — bare code, no re-expansion A(O) — A shortcuts to O
( ) new node branch — full embedding expansion L(Si) — Si branches off L
Decision flowchart
Start
Build embedding string for every superior host.
Is there only 1 superior host?
→ Yes: write its embedding string. Done.
Are all pairs coupled AND 4+ nodes?
→ Yes: write *{hosts in realm order}. Done.
Does a cycle exist?
→ Yes: find longest canonical cycle → write {cycle} with inline chords (X) and branches (embStr). Done.
No cycle
→ Find longest canonical chain → write nodes with branches (embStr). Done.
