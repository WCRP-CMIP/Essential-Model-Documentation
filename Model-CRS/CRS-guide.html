CRS — Canonical Realm String

A CRS is a short text string that uniquely describes how the components of an earth system model fit together. Two models with the same structure will always produce the same CRS. This page explains how to read one, and how to build one from scratch.

The eight component codes

Each part of a model gets a short code. Codes are always sorted in this order when constructing a string:

A atmosphere
Ac atmos. chemistry
Ae aerosol
L land surface
Li land ice
O ocean
Ob ocean biogeochem.
Si sea ice

Step 1 — Show which components live inside others

Some components are built directly into a host component and can't be separated from it — like aerosol physics coded into the atmosphere model. We call this embedding. In the CRS, an embedded component is written in square brackets immediately after its host, like a nested set of boxes.

The bracket colour matches the host. The code inside uses its own colour.

Ac embedded inside A

Atmospheric chemistry lives inside the atmosphere model.

A[Ac]
Ae inside Ac, Ac inside A

Aerosol lives inside atmospheric chemistry, which lives inside atmosphere. Three levels of nesting.

A[Ac[Ae]]
Ob and Si both inside O

Two components embedded in the same host. Both sit at the same level — listed in code order.

O[Ob][Si]
No embeddings

Some components stand alone — nothing inside them. The code is just written as-is.

L   Li

Step 2 — Show how the top-level components are connected

Components that exchange information with each other — passing heat, moisture, fluxes — are called coupled. Only the top-level "host" components can be coupled (not the ones embedded inside others). We use the coupling graph to determine the CRS coupling string, applying the first rule below that fits.

RULE A
One component, no coupling

If there is only one top-level component, just write its embedding string. Nothing else to do.

Example
CRS
A[Ac]
RULE B
A cycle — three or more components forming a ring

If the coupled components form a closed loop (every component passes to the next, and the last passes back to the first), wrap the whole sequence in curly braces { }. Use the longest loop you can find. Start from the component with the lowest code number (A before L before O, etc).

Triangle — A, L and O all coupled to each other
CRS — coupling only
{ALO}
CRS — with embeddings (CNRM-ESM2-1e)
{A[Ac][Ae]LO[Ob][Si]}
Chord — an extra direct link across the ring

Sometimes two components in a cycle are also directly connected, creating a shortcut. Write the shortcut target in (round brackets) immediately after the earlier component. Use only the bare code — no brackets needed since that component is already shown in full elsewhere in the string.

Cycle
A – L – O – Li – back to A
Extra edge
A and O are also directly linked
Position
A is at position 1 in the cycle, O is at position 3. Write (O) right after A
{A(O)LOLi}
RULE C
No cycle — a chain

If the coupled components don't form a closed loop, find the longest chain and write it in order. Any component connected to the chain but not part of it is written in (round brackets) after the chain node it connects to — with its full embedding string, since it only appears once.

Chain A – L – O, with Li branching off A
Chain
A – L – O (longest path)
Branch
Li connects to A → write (Li) after A
A(Li)LO
Star — A connects to L, O and Li, but they don't connect to each other
Chain
L – A – Li (longest, chosen by code order)
Branch
O connects to A → write (O) after A
LA(O)Li
RULE D — RARE
Full clique — every component coupled to every other (4 or more)

In practice this is very unlikely. If four or more components are all directly coupled to every other one, use an asterisk and curly braces: *{ }. A triangle (3 nodes, all connected) is handled by Rule B — only use *{ } for 4 or more nodes.

All 6 pairs coupled. List all components in code order.

*{ALLiO}
*{A[Ac[Ae]]LLiO[Ob][Si]}
↑ with embeddings added in

Notation at a glance

Symbol What it means Example
O[Ob] Ob is embedded inside O. Bracket colour = host colour. Ocean biogeochemistry lives inside the ocean model
{ } Cycle — the listed components form a closed ring of couplings {ALO}
(X) After a cycle node: a shortcut chord to X. After a chain node: X branches off here. {A(O)LOLi}
*{ } Every component in the list is directly coupled to every other (4+ only) *{ALLiO}

Real model examples

CNRM-ESM2-1e
{A[Ac][Ae]LO[Ob][Si]}
UKESM1 (K4 coupling)
*{A[Ac[Ae]]LLiO[Ob][Si]}
HadGEM3-GC31
{ALO[Si]}
4-cycle + chord
{A(O)LO[Ob][Si]Li}
Atmosphere only
A[Ac[Ae]]
Canonicality: because codes are always sorted in the fixed realm order and rules are applied in the same sequence, the same model structure always produces the same CRS string — regardless of the order you supply the embeddings and couplings.
